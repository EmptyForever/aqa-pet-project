# Мои тезисы по AQA

1. **Пирамида тестирования:** В идеале по теории - минимум UI тестов , большое кол-во интеграционных тестов, и самое большое кол-во юнит-тестов отвечающих за конретные задачи. Чем ниже по пирамиде тестирования , тем быстрее и дешевле тесты. Быстрее разрабатываются, быстрее выполняются. UI тесты более дорогие и долгие.

2. **Интеграционные тесты:** Проверяют работу взаимосвязи между модулями, например API, микросервис с микросервисом, бек с базой данных.

3. **UI-тесты:** UI тесты дольше всего выполняются, дольше всего разрабатываются и самые дорогие в поддержании.

4. **AQA vs Manual:** Manual проверяет руками. AQA тестирует кодом другой код. При изменениях в коде - автоматически проводит тестирование.

5. **End-to-End тест:** Проверка полного сценария использования продукта.

6.3. ## Selenium + Pytest: изученное

### Фикстуры (`@pytest.fixture`)
- Подготавливают ресурсы для теста.
- Передаются в тест по имени аргумента.
- Хранятся в `conftest.py` (доступны всем тестам).
- Не подставляются в классы автоматически.

### Декораторы
- `@pytest.fixture` — делает функцию фикстурой.
- `@pytest.mark.parametrize` — параметризация тестов.
- Декораторы изменяют поведение функции без изменения её кода.

### Локаторы (`By`)
- Способ найти элемент на странице.
- Используются: `By.ID`, `By.NAME`, `By.CLASS_NAME`, `By.XPATH`, `By.CSS_SELECTOR`.
- Хранятся как кортеж: `(By.ID, "user-name")`.
- При поиске распаковываются через `*`.

### Page Object Model
- Класс для страницы.
- В `__init__`: `driver` + локаторы.
- Методы: действия на странице.

### `self`
- Ссылка на текущий объект.
- Используется для сохранения данных внутри класса.

### `yield` в фикстурах
- Setup + teardown в одной функции.
- Код до `yield` — подготовка, после — очистка.
